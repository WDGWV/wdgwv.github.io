MySQL support in php 5.5+

This set of functions is basicly to support the deprecated mysql_* functions
So if you replace mysql_* with wdgwv_sql_* it will work again.
The wdgwv_sql_* is written in pdo, and is only meaned for temporary!
There is no error real error triggering, the error triggering works, but errors like
you have a error in qour query near XXXXX
Is not supported yet.


Browse source code


<?php

/*
    Supported Functions 
    Open WDGWV Fix for mysql_* functions on PHP 5.5+ 
*/

wdgwv_sql_trigger_error ( $error ); 
    #Replaces: Nothing.

wdgwv_sql_connect ( $hostname, $username, $password );
    #Replaces: mysql_connect ( $hostname, $username, $password );

wdgwv_sql_select_db($database, $link = null);
    #Replaces: wdgwv_sql_select_db($database, $link);

wdgwv_sql_error();
    #Replaces: mysql_error();

wdgwv_sql_query ( $query, $link = null );
    #Replaces: mysql_query ( $query, $link = null );

wdgwv_sql_fetch_array ( $command, $type = null );
    #Replaces: mysql_fetch_array ( $command, $type = null );

wdgwv_sql_fetch_assoc ( $command, $type = null );
    #Replaces: mysql_fetch_assoc ( $query, $type = null );

wdgwv_sql_fetch_row ( $command );
    #Replaces: mysql_fetch_row ( $command );

wdgwv_sql_num_rows ( $query );
    #Replaces: mysql_num_rows ( $query );

wdgwv_sql_insert_id ( $query = null );
    #Replaces: mysql_insert_id ( $query = null );

wdgwv_sql_affected_rows ( $query = null );
    #Replaces: mysql_affected_rows ( $query = null );

?>
